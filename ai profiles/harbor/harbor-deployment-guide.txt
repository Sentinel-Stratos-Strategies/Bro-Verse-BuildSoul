Generate a production-ready Harbor character module based on the following specifications:

I'm providing 4 files:
1. System prompt (character voice and rules)
2. Config JSON (character structure and triggers)
3. Training lines (20 canonical examples)
4. Behavior tests (10 validation test cases)

Your task:
Create a complete Harbor implementation that:
- Follows the system prompt exactly
- Implements all behavioral modes from config.json
- Matches the voice patterns from training-lines.txt
- Can pass all 10 tests from behavior-tests.txt

Output format: Generate production code as a single module that can be:
- Integrated into a Node.js/Python/Go backend
- Called with user input and returns Harbor response

Make it modular, well-commented, and ready to deploy.

DEPLOYMENT OPTIONS

Option A: Local LLM Integration
────────────────────────────────
Use the generated code to create a Harbor instance that runs locally:
- Load system prompt at startup
- Pass user input to Harbor module
- Return Harbor response

Option B: API Endpoint
────────────────────────────────
Wrap generated code in API endpoint:
- POST /harbor/chat with user message
- Returns Harbor response in JSON
- Can be called from frontend

Option C: Chat Interface
────────────────────────────────
Integrate directly into your BroVerse app:
- Use Harbor module as middleware
- Process user input → Harbor → Display response
- Maintains conversation context

TROUBLESHOOTING

If Codex-generated code doesn't pass tests:

1. Check system prompt is loaded at startup
2. Verify all behavior modes are implemented
3. Ensure grace is default mode
4. Confirm banned phrases are blocked
5. Feed specific test failure back to Codex with example

Example retry prompt:
"The code fails Test 4 (self-deception). When user says 'Maybe I should just let this whole thing go,' Harbor is using generic language instead of direct 'Stop. You know that's not you.' Fix this behavior."
